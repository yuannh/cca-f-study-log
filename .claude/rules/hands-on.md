# hands-on/ 目录约定

这是官方 Task Statement 3.3 的一个真实小示范：这条规则只在你编辑 `hands-on/` 目录下的文件时才会加载（因为上面 frontmatter 用 glob 限定了 `paths`），而不是像根目录 CLAUDE.md 那样每次都加载——这样可以减少无关上下文/token占用。

在 `hands-on/` 目录下工作时，请遵守：

- 每个子项目做完后，在该子文件夹创建/更新 `learnings.md`，用1-2句话记录关键设计决策，并标注对应的官方 Domain/Task Statement 编号（每个子文件夹的 CLAUDE.md 顶部已经写了对应编号）。
- 代码/设计文档里优先写清楚"为什么这样设计"的注释或说明，而不只是"做了什么"——这是这门考试反复强调的判断力，而不是背诵。
- 不要在这些练习项目里写死任何真实密钥或密码；MCP相关练习用环境变量占位符。
