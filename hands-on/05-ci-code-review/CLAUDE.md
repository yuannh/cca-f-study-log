# 动手项目5：Claude Code 接入 CI Pipeline

**对应计划**：第2周 周六（2026-07-18）| **官方 Domain**：3 Claude Code配置与工作流（Task Statement 3.6）

## 任务目标

把 Claude Code 接入一个 CI 流程（GitHub Actions 最方便），跑一次自动化代码评审，输出结构化、可被程序解析的结果。

## 验收标准

- [ ] 用 `-p`（非交互模式）运行，避免在 CI 里卡住等待输入
- [ ] 用 `--output-format json`（或 `--json-schema`）输出结构化评审结果，而不是纯文本
- [ ] CLAUDE.md 里写明了评审标准（哪些问题要报告、哪些属于风格问题可以跳过），而不是让 Claude 自由发挥
- [ ] 想办法验证：同一个 Claude 会话既生成代码又评审自己的代码，容易漏检——尝试用独立的评审 session/实例对比效果
- [ ] 如果做第二轮评审，测试是否能只报告"新增或仍未解决"的问题，避免重复评论

## 完成后

在本文件夹创建 `learnings.md`，记录你观察到的"自我评审"和"独立实例评审"效果差异。
