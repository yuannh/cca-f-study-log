# 动手项目8：人工复核 Workflow + 置信度校准设计

**对应计划**：第3周 周六（2026-07-25）| **官方 Domain**：5 上下文管理与可靠性（Task Statement 5.5、5.6）

## 任务目标

写一份设计文档（`design.md`），描述一个"模型输出 + 人工复核"的工作流：模型给每条抽取/判断结果打置信度分数，低置信度或有冲突的结果才路由给人工。

## 验收标准

- [ ] 说明为什么只看"整体准确率97%"这种聚合指标会掩盖某个文档类型/字段上的系统性差错，并设计按类型/字段拆分的准确率统计方式
- [ ] 设计一个分层随机抽样方案：即使是"高置信度"的结果也要抽样人工检查，用来发现新的错误模式
- [ ] 说明置信度阈值应该用带标签的验证集校准，而不是拍脑袋定一个数字
- [ ] 如果涉及多来源信息冲突（比如两份可信来源给出不同的统计数字），设计一个"保留双方数值+标注来源+交给下游/人工判断"的处理方式，而不是自己选一个"看起来更对"的数字

## 完成后

把 `design.md` 留在本文件夹作为考前复习笔记。
