# 动手项目1：最小 Agentic Loop

**对应计划**：第1周 周五（2026-07-10）| **官方 Domain**：1 智能体架构与编排（Task Statement 1.1）

## 任务目标

用 Claude Agent SDK（或直接调用 Messages API + tool_use）实现一个最小但正确的 agentic loop：

1. 发送请求给 Claude，检查响应的 `stop_reason`
2. 如果是 `tool_use`：执行被请求的工具，把结果作为新的一轮消息追加回对话历史，再次发送请求
3. 如果是 `end_turn`：结束循环，返回最终答案

## 验收标准（对照官方考点自查）

- [ ] 循环的终止条件是检查 `stop_reason == "end_turn"`，而不是解析文本内容、也不是设一个固定的最大轮数当作主要终止条件
- [ ] 每一轮工具执行结果都被正确追加到对话历史（而不是丢弃或覆盖）
- [ ] 工具至少有2个，且是真实可执行的（例如查天气、算数、读文件），不是 mock 占位
- [ ] 能在日志里清楚看到每一轮 Claude 的决策过程
