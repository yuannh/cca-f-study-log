## 关键架构判断（考前秒读）

1. **上下文不自动继承 → coordinator 必须显式传。**
   每个 subagent 是独立调用，只看到被喂进 prompt 的东西。分析员看不到搜索员的对话，
   所以 coordinator 必须把搜索结果"提炼后显式塞进"分析员的 prompt——不能指望它自动继承。

2. **看依赖决定并行还是串行。**
   两个搜索互不依赖 → 同一轮**并发派生**（省时间）；分析依赖搜索结果 → 必须**串行**（等搜索收齐再派）。

3. **subagent 通信只过 coordinator，不直连。**
   coordinator 是唯一枢纽：所有结果先回到它、再由它转发。subagent 之间开直连是反模式。

4. **失败要如实回传、带部分结果，不崩不编。**
   实测验证：一个搜索员失败时，coordinator 记录失败 + 用"资料缺失"占位继续，分析员诚实标注
   "无法完整对比"。对比之下——崩掉整个流程、或假装成功编造数据，都是反模式。

## 亲眼验证到的一点

"故意失败"那次运行最有价值：报告没有崩、也没有对一台根本没查到的电脑凭空编对比，而是
**诚实降级 + 标注缺口**。这让"错误如实传播"从一个概念，变成了见过的事实。
