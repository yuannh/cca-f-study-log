# 动手项目6：结构化数据抽取 Pipeline

**对应计划**：第3周 周二（2026-07-21）| **官方 Domain**：4 提示工程与结构化输出（Task Statement 4.3、4.4）

## 任务目标

找几份格式不完全统一的"文档"（发票、简历、会议纪要都行），用 `tool_use` + JSON schema 从中抽取结构化字段。

## 验收标准

- [ ] 用工具定义（而不是让模型直接吐 JSON 文本）来强制输出结构，字段里可能缺失的信息设为 nullable/optional，而不是强迫模型瞎编一个值
- [ ] 实现一个 validation-retry 循环：把校验失败的具体错误信息（不是笼统的"格式不对"）附加到重试请求里
- [ ] 故意测试一个"文档里根本没有这个信息"的场景，验证重试对这种情况是否真的没用（这是官方明确的考点）
- [ ] 如果同一批文档有多种格式（比如有的用表格、有的用自然语言描述），至少提供2-3个 few-shot 示例来提升泛化

## 完成后

在本文件夹创建 `learnings.md`，写下你遇到的哪类错误"重试有效"、哪类错误"重试无效"，各举一个具体例子。
